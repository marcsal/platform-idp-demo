Internal Developer Platform (IDP)

Por Marcsal - Platform Engineer

Esta página demuestra la construcción de un "Golden Path" para despliegues. Es una prueba de concepto E2E alojada en Google Kubernetes Engine (GKE), diseñada para reducir la carga cognitiva de los equipos de desarrollo mediante automatización y GitOps.

1. Infraestructura Base Eficiente

Aprovisionamiento automatizado del clúster mediante Terraform. Se ha aplicado una estrategia FinOps separando el plano de control y utilizando un pool de nodos preemptibles para optimizar el coste de los entornos de desarrollo.

Terraform GKE FinOps

2. Developer Experience (DevEx) & GitOps

El ciclo de vida del software está gobernado por ArgoCD. Los desarrolladores solo necesitan hacer push a su repositorio; el modelo Pull asegura que el estado deseado en Git se reconcilie automáticamente en el clúster de Kubernetes, eliminando la necesidad de dar acceso directo a la infraestructura.

ArgoCD Kubernetes CI/CD

3. Telemetría y Enrutamiento Centralizado

La plataforma ofrece observabilidad "out-of-the-box" mediante el stack de Prometheus y Grafana. El tráfico externo se gestiona de forma eficiente a través de un único NGINX Ingress Controller, optimizando el uso de balanceadores de carga en la nube.
